드디어 JBank 과제를 해내었다. 나름대로 얻은 것이 많다.
Calendar와 SimpleDateFormat클래스를 이용해서 현재 날짜를 표시하는 방법을
알았고, printf로 %를 표시할 때는 %%라고 해 주어야 한다는 것도 알았다.
  그리고 static에 대해 더 잘 알게 되었는데, static 필드를 초기화할 때는 반드시
static 초기화 블록에 해 주어야 한다는 것을 알았다. 생성자에다가 해 주어도
상관은 없는데, 그렇게 되면 static만의 공유한다는 의미가 사라져서 객체가 새로
생성될 때마다 static 필드 값이 바뀐다.
  또한 static 필드만 있는 메서드는 static을 쓰는 게 좋다. 인스턴스 메서드로
선언하면 static을 다루는 의미가 없기 때문이다.
  그리고 그 동안 크게 착각하던 게 있었는데 this나 클래스 이름을 생략할 수 있는
것은 오로지 생성자나 초기화 블록 안에서만 가능하다는 것이다. 메서드 안에서도
생략해서 쓰면 프로그램은 무엇이 매개변수이고 무엇이 필드인지 구분을 못 한다.
예를 들어,

public void foo(String name){
	name = name;
}

이런 건 안 된다.

  아 참고로 메인 클래스 내에 프라이빗 제한자 설정이 안 되는 문제는 사실 메인
'클래스'가 아니라 메인 '메서드'에서 안 되는 거였다. 왜지????
  그리고 \t은 단순히 탭만큼 칸을 띄우는 게 아닌 듯 하다. 옆에 있는 문자열의
길이에 따라 다르게 조절이 되기 때문이다. 이것도 좀 더 알아봐야할 문제다.
  
  어쨌든 재미있고 유용한 과제였다.
